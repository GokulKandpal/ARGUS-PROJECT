<h1>ARGUS Project</h1>

<h2>Project Overview</h2>

ARGUS is an intelligence and surveillance platform developed by INNEFU. The application is used for monitoring, search operations, alert handling, investigations, and case management activities.<br>
This project involved validating application functionality and ensuring all modules worked according to business requirements through manual and automation testing.<br><br>

<h2>Project Objective</h2>

The main objective of this project was to test application functionality, identify defects, ensure system stability, and improve overall product quality before deployment.<br><br>

<h2>Modules Covered</h2>

- Login and Authentication<br>
- Dashboard<br>
- Alert Management<br>
- Search Functionality<br>
- Case Management<br>
- Reports<br>
- Snapshot Module<br>
- User Management<br>
- Role Based Access Control<br>
- System Configuration<br><br>

<h2>Testing Activities Performed</h2>

Manual Testing:<br>
- Functional Testing<br>
- Smoke Testing<br>
- Regression Testing<br>
- UI Testing<br>
- Integration Testing<br>
- Retesting<br><br>

Automation Testing:<br>
- Login scenarios automation<br>
- Search workflow automation<br>
- Form validation scenarios<br>
- Regression suite execution<br>
- Alert workflow scenarios<br><br>

<h2>Tools and Technologies Used</h2>

Testing Tool: Katalon Studio<br>
Automation Tool: UI.Vision RPA<br>
Defect Tracking Tool: Jira<br>
Version Control: Git<br>
Documentation: Excel and Reports<br><br>

3. Test Case Creation<br>
4. Test Execution<br>
5. Defect Logging<br>
6. Retesting<br>

<h2>Defect Lifecycle</h2>
